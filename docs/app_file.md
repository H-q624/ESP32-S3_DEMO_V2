# File (SD卡存储) 组件

## 职责

SD卡文件系统管理与IMU/音频数据持久化存储，采用双缓冲/队列架构平衡采集与写入速度差异。

## 核心设计

### 类层次

```
SDcard (基类)
├── 静态资源管理 (卡句柄、互斥锁)
├── 通用文件操作 (打开/写入/关闭)
└── 纯虚接口 (子类必须实现)
        │
        ├── IMU_Data (派生)
        │   ├── 双缓冲机制
        │   ├── 双任务模型 (采集+写入)
        │   └── CSV格式存储
        │
        └── MIC_Data (派生)
            ├── 队列接收WAV包
            ├── 双任务模型 (采集+写入)
            └── WAV流式写入
```

### IMU双缓冲架构

```
┌─────────────────────────────────────────┐
│           采集任务 (Priority 5)          │
│  ┌─────────────┐    ┌─────────────┐     │
│  │  Buffer 0   │◄──►│  Buffer 1   │     │
│  │  100样本    │    │  100样本    │     │
│  └──────┬──────┘    └─────────────┘     │
│         │ 满/超时(500ms)                 │
│         ▼                                │
│    ┌─────────┐                           │
│    │ Queue   │ ───────────────────────┐  │
│    │ (深度2) │                        │  │
│    └─────────┘                        │  │
└─────────────────────────────────────────┘
                                          │
┌─────────────────────────────────────────┤
│           写入任务 (Priority 2)          │
│         ▼                               │
│    ┌─────────┐                          │
│    │ Queue   │◄─────────────────────────┘
│    └────┬────┘
│         │ 批量写入
│         ▼
│    ┌─────────┐
│    │ SD Card │
│    │ 5MB/文件│
│    └─────────┘
└─────────────────────────────────────────┘
```

### MIC流式写入架构

```
┌─────────────────────────────────────────┐
│        MIC采集任务 (Priority 4)          │
│  I2S → 24bit→16bit → 累积50ms → WAV包  │
│                              ▼          │
│                      ┌─────────┐         │
│                      │ wav_queue│        │
│                      │ (深度20) │─────────┐
│                      └─────────┘         │
└─────────────────────────────────────────┘
                                          │
┌─────────────────────────────────────────┤
│        MIC写入任务 (Priority 2)          │
│         ▼                               │
│    ┌─────────┐                          │
│    │ wav_queue│◄────────────────────────┘
│    └────┬────┘
│         │ 流式写入WAV
│         ▼
│    ┌─────────┐
│    │ SD Card │
│    │ 5MB/文件│
│    └─────────┘
```

### 关键参数

| 模块 | 参数 | 值 | 说明 |
|------|------|-----|------|
| IMU | 缓冲区数 | 2 | 双缓冲 |
| IMU | 每缓冲样本 | 100 | 约1秒数据@100Hz |
| IMU | 切换超时 | 500ms | 最大缓冲延迟 |
| IMU | 写入队列深度 | 2 | 匹配缓冲区数 |
| MIC | WAV包大小 | 50ms | 800采样点 |
| MIC | WAV队列深度 | 20 | 约1秒缓冲 |
| 通用 | 单文件上限 | 5MB | 约10分钟IMU或2.7分钟音频 |

### 线程安全

- **静态互斥锁**: `s_mutex` 保护所有文件操作
- **优先级**: 采集任务(5/4) > 写入任务(2)，确保SD卡阻塞不影响数据采集

### 文件格式

**IMU (CSV)**:
```
timestamp,accel_x,accel_y,accel_z,gyro_x,gyro_y,gyro_z,roll,pitch,yaw,combined_accel,combined_gyro,pre_judge,btn_sign
```

**MIC (WAV)**:
- 16-bit PCM, 16kHz, Mono
- 文件头44字节，数据流式追加

### 关键接口

**SDcard基类**:
- `app_file_init_once()`: 静态方法，确保单次初始化
- `app_file_open_file()`: 创建新文件

**IMU_Data**:
- `imu_data_init()`: 初始化双缓冲、队列、文件
- `collection_task_loop()`: 高优先级采集循环
- `write_task_loop()`: 低优先级写入循环

**MIC_Data**:
- `mic_data_init()`: 初始化
- `mic_write_task_loop()`: WAV流式写入循环
- `write_wav_packet()`: 写入单个WAV包
- `update_wav_header()`: 关闭文件时更新WAV头

## 依赖

- SPI: `CONFIG_SPI_PIN_*`
- IMU队列: `data_queue` (接收MPU数据)
- MIC队列: `wav_queue` (接收WAV包)
- 模式: `get_current_data_mode()` (控制启停)
- 事件组: `mode_switch_event_group` (切换同步)
