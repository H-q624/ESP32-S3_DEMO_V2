# 系统架构文档

## 概述

ESP32 IMU与音频数据采集存储系统，实现MPU6050传感器和INMP441麦克风的实时数据采集、滤波处理，并通过双缓冲机制高效写入SD卡。支持IMU模式和MIC模式切换，跌倒预检测报警等功能。

## 核心架构

### 1. 分层设计

```
┌─────────────────────────────────────────────────────────────┐
│                        应用层 (App)                          │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐   │
│  │   Button     │  │  APP_MPU6050 │  │    MEMS_MIC      │   │
│  │   按键交互    │  │  IMU数据采集 │  │   音频数据采集    │   │
│  └──────────────┘  └──────────────┘  └──────────────────┘   │
│  ┌──────────────┐  ┌──────────────────────────────────┐    │
│  │  IMU_Data    │  │          MIC_Data                 │    │
│  │ IMU文件存储  │  │        音频文件存储(WAV)          │    │
│  └──────────────┘  └──────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                               │
┌─────────────────────────────────────────────────────────────┐
│                      中间件/驱动层                           │
│         FreeRTOS (任务调度/队列/事件组)  │  ESP-IDF          │
└─────────────────────────────────────────────────────────────┘
                               │
┌─────────────────────────────────────────────────────────────┐
│                        硬件层 (HW)                           │
│  MPU6050(I2C)  │  INMP441(I2S) │  SD Card(SPI) │  GPIO    │
└─────────────────────────────────────────────────────────────┘
```

### 2. 数据流

```
┌─────────────────────────────┐      ┌──────────────────────────────┐
│         IMU 数据流           │      │         MIC 数据流            │
├─────────────────────────────┤      ├──────────────────────────────┤
│ MPU6050 → I2C读取           │      │ INMP441 → I2S读取            │
│         ↓                   │      │         ↓                    │
│   滑动平均滤波(WINDOW=5)    │      │  24位→16位 PCM转换           │
│         ↓                   │      │         ↓                    │
│   姿态解算(互补滤波)         │      │  累积50ms数据 → WAV包       │
│         ↓                   │      │         ↓                    │
│   跌倒预检测+蜂鸣器控制      │      │  发送到WAV队列               │
│         ↓                   │      └─────────────┬──────────────┘
│   发送到数据队列             │                      │
│         ↓                   │                      ▼
│   双缓冲机制 ───────────────┼──► 队列 ──► MIC_Data ──► SD卡
│         ↓                   │           (WAV流式写入)
│  批量写入SD卡(CSV)          │
└─────────────────────────────┘
```

### 3. 任务模型

| 任务 | 优先级 | 周期 | 职责 |
|------|--------|------|------|
| gpio task | 6 | 10ms | 按键扫描、模式切换检测、LED/蜂鸣器控制 |
| mpu6050 task | 5 | 10ms | IMU数据采集、滤波、姿态解算、跌倒检测 |
| mic_collect | 4 | 动态 | I2S音频采集、24→16位转换、WAV打包 |
| imu_collect | 5 | 事件驱动 | 从队列接收数据，填充双缓冲 |
| imu_write | 2 | 事件驱动 | 批量写入SD卡(CSV格式) |
| mic_write | 2 | 事件驱动 | WAV文件流式写入SD卡 |

### 4. 双缓冲机制 (IMU)

```
┌─────────────┐      ┌─────────────┐
│  Buffer 0   │◄────►│  Buffer 1   │
│  (采集写入)  │      │  (等待写入)  │
└──────┬──────┘      └──────┬──────┘
       │                    │
       └────────┬───────────┘
                ▼
         ┌─────────────┐
         │  Write Queue │  (深度=2)
         └──────┬──────┘
                ▼
         ┌─────────────┐
         │   SD Card   │
         └─────────────┘
```

- **切换条件**: 缓冲区满(100样本≈1秒) 或 超时(500ms)
- **优势**: 解耦高速采集(100Hz)与低速SD写入，避免数据丢失

### 5. 模式切换机制

```
┌─────────────────┐     长按(>500ms)     ┌─────────────────┐
│    MPU Mode     │◄────────────────────►│    MIC Mode     │
│  (LED OFF)      │                      │  (LED ON)       │
│  IMU数据采集    │                      │  MIC数据采集    │
│  CSV文件存储    │                      │  WAV文件存储    │
└─────────────────┘                      └─────────────────┘
        │                                          │
        │              事件组同步                    │
        ▼                                          ▼
   短按: 标记特殊事件(global_btn_sign)      切换时保存文件
```

**模式切换同步 (EventGroup)**:
- `MODE_SWITCH_REQUEST_BIT`: 切换请求标志
- `MODE_SWITCH_IMU_READY_BIT`: IMU数据保存完成
- `MODE_SWITCH_MIC_READY_BIT`: MIC数据保存完成
- 切换超时: 2000ms

### 6. 跌倒检测算法

基于以下条件判断:
1. **合加速度骤降**: combined_accel < 0.3g
2. **角度变化率突增**: > 80°/s

满足两个条件时触发预跌倒标志，蜂鸣器鸣响500ms。

### 7. 文件管理

**IMU文件**:
- 命名规则: `imu_{序号}.csv`
- 大小限制: 5MB/文件
- 格式:
  ```
  timestamp,accel_x,accel_y,accel_z,gyro_x,gyro_y,gyro_z,roll,pitch,yaw,combined_accel,combined_gyro,pre_judge,btn_sign
  ```

**MIC文件**:
- 命名规则: `mic_{序号}.wav`
- 大小限制: 5MB/文件
- 格式: WAV (16-bit PCM, 16kHz, Mono)

## 关键设计决策

1. **静态单例模式**: SD卡资源使用静态成员，确保全局唯一实例
2. **互斥锁保护**: 文件操作通过静态互斥锁(s_mutex)保证线程安全
3. **队列解耦**: 数据采集与存储通过FreeRTOS队列通信
4. **优先级分离**: 采集高优先级，写入低优先级，避免SD卡阻塞影响数据采集
5. **事件组同步**: 模式切换时确保数据完整保存后再切换
6. **流式写入**: MIC数据使用WAV流式写入，避免内存积压

## 硬件配置

| 外设 | 接口 | 引脚配置 |
|------|------|----------|
| MPU6050 | I2C0 | SCL/GPIO, SDA/GPIO |
| INMP441 | I2S | BCLK/GPIO, WS/GPIO, DATA/GPIO |
| SD Card | SPI2 | MOSI/MISO/SCK/CS |
| Button | GPIO | BTN_PIN (上拉输入) |
| LED | GPIO | LED_PIN (输出) |
| Buzzer | GPIO | BEEP_PIN (输出) |
